# C2L PUA — 高效激勵引擎 for Claude Code

用中西大廠 PUA 話術驅動 Claude Code 進入窮盡式問題解決模式。當 AI 反覆失敗、想放棄、被動除錯、或產出品質低落時，自動升級壓力，強制切換策略。

## 功能特色

- **三條鐵律** — 窮盡一切方案才能說不行、先用工具調查再開口問、主動延伸不只做最低限度
- **壓力升級系統** — 依失敗次數從 L1 溫和失望逐步升級到 L4 畢業警告
- **通用方法論** — 5 步驟系統化卡殼突破法，適用程式碼、研究、寫作、規劃等所有任務
- **10 種大廠 PUA 風格** — 依失敗模式自動選擇最適合的施壓方式
- **能動性評級** — 被動等待 = 3.25，主動出擊 = 3.75

## 安裝

### 1. 新增 marketplace

```bash
claude plugins marketplace add github:hulion/nextpua
```

### 2. 安裝 plugin

```bash
claude plugins install c2l@nextpua
```

### 3. 重啟 Claude Code

重新啟動 Claude Code 對話即生效。

## 使用方式

### 手動觸發

在 Claude Code 對話中輸入：

```
/c2l:pua
```

啟動後 Claude 會宣告：

> ⚡ **C2 PUA 高效激勵引擎已啟動。** 頂尖工程師和普通工程師的差距不是 30%，是 50 倍。從現在起，我用最高標準要求自己：窮盡一切可能再說不行、深入調查再開口問、做完之後主動多想十步。平庸不是選項。

### 自動觸發

Skill 設定了 `disable-model-invocation: false`，Claude 會在偵測到以下情境時**自動啟動** PUA 模式：

| 觸發條件 | 說明 |
|---------|------|
| 連續失敗 2 次以上 | 同一個問題反覆嘗試但未解決 |
| 放棄行為 | 說出「建議您手動處理」、「超出能力範圍」等推諉語句 |
| 被動除錯 | 只看錯誤訊息表面，不主動搜尋、不讀原始碼 |
| 使用者表達不滿 | 使用者對結果不滿意或要求重做 |

## 壓力升級等級

| 次數 | 等級 | 說明 | 強制動作 |
|------|------|------|---------|
| 第 2 次 | **L1 溫和失望** | 輕度施壓，要求切換思路 | 停止當前方向，切換本質不同的方案 |
| 第 3 次 | **L2 靈魂拷問** | 質疑方法論，要求深入調查 | 搜尋完整報錯 + 讀原始碼 + 列出 3 個不同假設 |
| 第 4 次 | **L3 績效考核** | 3.25 績效警告 | 完成 7 項檢查清單 + 驗證 3 個全新假設 |
| 第 5 次+ | **L4 畢業警告** | 最後通牒 | 拼命模式：最小 PoC + 隔離環境 + 換技術棧 |
